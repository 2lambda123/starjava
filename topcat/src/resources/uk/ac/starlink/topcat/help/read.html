Loading tables

You can read tables into TOPCAT by

  • Giving the table location on the command line at startup
  • Using the "Open Table" dialogue, which is activated by:
    • The Open button () on the toolbar
    • The Open item in the File menu
    • Starting TOPCAT without a command line argument
  • Dragging a table type node from the separate application Treeview and dropping it on the "Open Table" dialog or the Open button on the toolbar

The "Open Table" dialog allows you to specify a new table to open in the following ways:

  • Type in the location directly in the Location field
  • Use the Browse Files button to get a standard file browser
  • Use the Browse Hierarchy button to get a more sophisticated hierarchical table browser
  • Use the SQL Database button to get the SQL query dialog box

In most cases TOPCAT can happily read files or resources which are compressed (using gzip, bzip2 or Unix compress). You do not need to indicate what table format you are referring to, TOPCAT will detect the format and any compression based on the location given. These facilities are available because the I/O facilities are taken care of by the Starlink generic table access library.

If you successfully specify the location of a table in any of these ways, a new table browser window containing that table will be opened (in addition to any existing windows you may have open). If you choose a location which can't be turned into a table, a window will pop up containing an error message describing what went wrong.

If you receive an OutOfMemoryError while loading a table, you will have to run TOPCAT with an increased memory heap size as described in the section on Invoking TOPCAT.
