Supported data formats

SPLAT can read and write spectra stored in four fundamental formats, Starlink NDF & NDX, FITS and simple TEXT. NDFs are stored in container files that have the extension .sdf. FITS files have one of the extensions .fits or .fit and TEXT files .txt or .lis. NDX format spectra, when stored on disk, will have extension .xml.

Line identifier catalogues currently have one format - TEXT based in files with extension .ids.

Note that although there are a set of default file extensions you can select the type that a spectrum is opened as using the File=>Open dialog in the main window. Currently it isn't possible to do this from the command-line.

Spectra stored in TEXT files must have the simple format of two or three columns of data, separated by whitespace (that is spaces or tabs). Any comments in the file must be indicated by a hash sign, blank lines are also allowed. The three columns should have the following values: 

  wavelength count [error-in-count]
The error column being optional.

The format of line identification catalogues is like that of spectral files, except the second column contains a string that identifies the line (with no spaces, i.e. H-alpha not H alpha). It is also possible to define the system of the coordinate values in a header section, see the SPLAT builtin catalogues for examples of this.

NDFs stored within container files can be accessed, but only by entering the full HDS path on the command-line or in the File=>Open dialog. FITS extensions may also be open in this fashion (by appending the extension number in square brackets).

Internally all these formats are stored as double precision coordinates and data values. SPLAT only requires that coordinates increase or decrease monotonically.
